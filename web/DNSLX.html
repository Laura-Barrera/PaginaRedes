SERVIDOR DNS

DNS es una tecnología esencial en cualquier servidor Linux, incluyendo Debian 11. Es responsable de traducir nombres de dominio fáciles de recordar (como "google.com") en direcciones IP (como "216.58.194.174"), lo que permite que los dispositivos se comuniquen entre sí en la red.

En este caso el DNS se implementa para que los equipos que se conecten al servidor puedan acceder a servicios como FTPs, HTTP y SMTP haciendo uso del dominio del servidor.


Entrar a la terminal de linux y logearse como super usuario con el comando su -l. Luego con el comando apt-get install bind9 instalar los paquetes necesarios para permitir la instalación del DNS.

Entrar a la ubicación /etc/bind/ y ubicar los archivos de zona directa y zona inversa.


Hacer una copia de los archivos identificados con el comando cp db.local nuevo archivo y validar que se hayan creado correctamente.

Dentro del archivo de zona directa creado, configurar el dominio y nombre del servidor y luego la IP que redirecciona al mismo servidor linux.


Dentro del archivo de zona inversa creado, configurar el dominio y nombre del servidor linux.

Dentro de la misma carpera /etc/bind/ acceder al archivo named.conf.local y configure la zona directa con el dominio del servidor y la zona inversa con la ip inversa de dominio.


Finamente, acceder al archivo /etc/resolv.conf y definir en el domain, y search el dominio del servidor y en el nameserver la ip del equipo.
