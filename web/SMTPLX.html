SERVIDOR SMTP

SMTP es un protocolo utilizado para el envío de correos electrónicos desde un servidor a otro servidor o cliente de correo electrónico.

En un servidor Linux, SMTP es utilizado por el servidor de correo electrónico para enviar correos electrónicos salientes. El servidor de correo electrónico recibe los correos electrónicos a través del protocolo POP3 o IMAP y los entrega a los clientes de correo electrónico. Además, los servidores de correo electrónico utilizan el protocolo SMTP para comunicarse entre sí y entregar los correos electrónicos a sus destinatarios.


Entrar a la terminal de linux y logearse como super usuario con el comando su -l. Luego con el comando apt-get install postfix se descargan los archivos para el servicio de correo electrónico en Linux server.

Automaticamente se ejectutará el programa de configuración de postfix, dar clic en Aceptar


Seleccionar que el servicio de correo electónico funcione como un Sitio de Internet para que este funcione con la IP y el dominio del Linux server.

Finalmente, se configura el dominio de uso en el servicio de correo electrónico y se da clic en Aceptar.


Desde la terminar como super usuario, ejecutar el comando apt-get install dovecot-core dovecot-pop3d para instalar el protocolo POP3 que permite el funcionamiento del servicio de correo instalado con postfix.

Acceder al archivo /etc/postfix/main.cf y configurar la linea de compatibility_level = 3.6.


Dentro del mismo archivo /etc/postfix/main.cf verificar que las configuraciones del servidor coincidan con el dominio configurado en postfix.

Acceder al archivo /etc/dovecot/conf.d/10-ssl.conf y cambiar que la linea sss = no y validar que las lineas pertienentes esten descomentadas.


Acceder al archivo /etc/dovecot/conf.d/10-auth.conf y modificar la linea disable_plaintext_auth = no y validar que las otras lineas esten descomentadas y su valor coincida.

Acceder al archivo /etc/dovecot/conf.d/10-mail.conf y validar que las lineas pertinentes esten descomentadas y su valor coincida segun la imagen.


Dentro del mismo archivo /etc/dovecot/conf.d/10-mail.conf validar que la linea mail_privileged_group = mail y que las lineas descomentadas coincidan.

Finalmente, reiniciar el servidor SMTP y el servicio POP3 con los comandos service postfix restart y service dovecot restart y verificar que su estado sea activo.
